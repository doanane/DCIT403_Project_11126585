    | ALTERNATIVE SEVERITY DETERMINATION
    | (sub-plan within assess_incident)
    |
    +-- Sensor type == chemical:
    |     value > 200 -> score 4 (critical)
    |     value > 100 -> score 3 (high)
    |     value > 50  -> score 2 (medium)
    |     else        -> score 1 (low)
    |
    +-- Sensor type == temperature:
    |     value > 120 -> score 4 (critical)
    |     value > 80  -> score 3 (high)
    |     value > 50  -> score 2 (medium)
    |     else        -> score 1 (low)
    |
    +-- Sensor type == air_quality / radiation:
    |     ratio > 3.0 -> score 4 (critical)
    |     ratio > 2.0 -> score 3 (high)
    |     else        -> score 2 (medium)
    |
    +-- Reports >= 2 (no sensor):
          score at least 2 (medium)

CONTEXT CONDITIONS FOR PLAN SELECTION
  The assess_incident plan internally branches:
    - _determine_waste_type applies priority-ordered rules
    - _determine_severity applies threshold comparison per sensor type
  These internal branches implement Prometheus alternative plan
  selection based on context conditions (Chapter 9.2.1)
